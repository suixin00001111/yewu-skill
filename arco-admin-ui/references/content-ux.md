# ui组业务组件 · 内容区 UX（优先工作台）

> **范围**：只优化内容区观感与信息结构。  
> **禁止改壳**：顶栏 `#004EA2`、白侧栏、菜单选中 `#E6F1FF`、主按钮 `#165DFF` 以 [product-shell.md](product-shell.md) 为准，本文件不得覆盖。  
> **参考图**：`examples/workbench.png`（基础）、`examples/workbench-ops.png`（运营密度增强，**只学内容区布局与节奏，不学其顶栏/主题色**）。

生成 **工作台** 时本文件 **必读**。

---

## 0. 优先级

| 级别 | 内容 | 冲突时 |
|------|------|--------|
| L0 | 产品壳顶栏/菜单/主色 | 最高，不可改 |
| L1 | 既有分块、页头操作、表格浅灰头 | 保持 |
| L2 | 本文工作台内容区质感 | 在 L0/L1 内细化 |

配色说明：参考图若为青绿/白顶栏等，**映射**到本规范：强调色用 `#165DFF`，浅底用 `#E8F3FF` / `#F2F3F5` / `#F7F8FA`，成功/警告/危险用 tokens 语义色，**不要**整页换成参考图主题色。

## 1. 工作台总结构（推荐）

```
内容区浅灰底 #F2F3F5
┌─ 可选顶工具条：业务 Tab/标签筛选 · 右侧搜索 ─────────────────┐
│ 左主栏 ~68–74%                          │ 右辅栏 ~26–32%   │
│  A 待办/告警横幅卡（可含步骤条）          │  R1 上下文卡     │
│  B 数据概览：指标砖 + 趋势图              │  R2 常用功能宫格 │
│  C 业务动态/巡检/消息区                   │  R3 变更日历/日程│
│  D 列表/表格模块（Tab + 浅灰表头 Table）  │                 │
└─────────────────────────────────────────┴─────────────────┘
```

### 布局原则

- 桌面 **左右两栏**；`<1200` 可改为单列（右栏下移）。
- **多白卡片分区**，卡片间距 **16px** 统一；圆角 **6–8px**；轻边框 `#E5E6EB`。
- 信息密度 **中高**（参考 workbench-ops）：指标可扫读、列表完整，但 **禁止** 营销大英雄区、大插画 Banner、紫渐变玻璃卡。
- 左栏 **3–5** 模块，右栏 **2–3** 模块；模块可按业务改名，但保持「概览 → 动态 → 列表」节奏。
- 图标统一 **IconPark** outline/彩色面性小图标（同页风格一致）。

## 2. 卡片通用

### 2.1 头

- 左：可选 **3–4px 主色竖条** + 标题 16px / 600 / `color-text-1`
- 中/旁：可选「更新时间」12–13px `color-text-3`
- 右：`查看更多` 等 **文字链** `#165DFF`，勿每卡放实心主按钮
- 头底浅分割线 `#E5E6EB`

### 2.2 体

- 内边距 **16 / 20**
- 正文 14 / 辅助 12–13；四级灰用满
- 行悬停 `color-fill-1`；空态 `Empty`

## 3. 模块规范（对标优秀业务工作台）

### 3.1 内容区顶工具条（可选）

- 左：业务对象 **Tab 或 标签组**（如分类/产品线）；当前项主色底白字或主色字+底划线
- 右：搜索框（placeholder 业务文案），高度 small/default
- 不要做成第二套导航顶栏（高度克制，白/浅底即可）

### 3.2 待办 / 告警横幅卡（强烈推荐有「需处理事项」时）

参考 workbench-ops 上区：

- 浅主色/浅填充背景（如 `#E8F3FF` 或 `primary-1`），**非**大图营销头
- 标题一行（事件说明）+ 辅助说明 1–2 行
- **一个**主操作按钮（如「去处理」「同意」）
- 可选底部 **横向步骤** 1–4 步：数字圆点 + 文案 + 连接线（与详情 Steps 气质一致，节点+连线）
- 可选右侧轻装饰插画/图表，不抢主文案

### 3.3 数据概览（指标砖 + 趋势）

- **指标砖**：2×3 或 3×2 浅底卡片网格；每砖：指标名、大号数值、单位、可选「较昨日」涨跌（绿升红降或本规范 success/danger）
- **趋势图**：与指标同卡或并排；折线主色 `#165DFF`；tooltip 简洁；高度约 180–240
- 指标字号：数值 20–24 半粗，名称 12–13 辅助色
- 禁止首屏四个巨型海报 Statistic 占满无业务上下文

### 3.4 业务动态 / 巡检 / 通知条

- 可含 **信息条**（浅底 + 文案 + 文内链接「点击查看」）
- 双列或四格 **汇总指标**；可用 **进度条** 表达占比（主色轨道）
- 地点/对象名 + 辅助地址一行

### 3.5 列表 / 表格模块

- 卡内 **Tab** 切换数据视角
- 标准 **Table**：浅灰表头、斑马可选、操作列文字链；状态 **Tag**
- 底部分页若数据多；工作台内嵌表默认 5–10 行 +「查看更多」

### 3.6 右栏 · 上下文卡

- 当前对象标题 + 位置/副标题
- 状态 **Tag** 组（多色浅底：运行中/处理中/停运等）
- 迷你四宫格计数（新增/变更/…）
- 底部一行说明 + **一个**主按钮（如「去新增」）

### 3.7 右栏 · 常用功能

- **3×2 或 3×3 宫格**
- 每项：彩色圆角图标底（可用多色面，但克制、非霓虹）+ 文案；图标用 **IconPark**
- 比「快捷操作」更偏入口导航；左栏也可放一排快捷，右栏常用功能更紧凑

### 3.8 右栏 · 变更日历 / 日程

- 月历：选中日主色底白字；图例色点（新增/变更/…）用语义色
- 下方事件列表：类型 Tag + 标题 + 摘要 + 「去查看」链接
- 分组标题可折叠「展开更多」

### 3.9 传统模块仍可用

快捷操作、我的任务、待办 Tab 列表、通知公告、用户卡 —— 与上列可组合，**按业务裁剪**，不必堆满所有模块。

## 4. 视觉细节（更好看的关键）

| 点 | 做法 |
|----|------|
| 层级 | 白卡浮在浅灰底；卡内再用浅灰砖做指标，形成两层表面 |
| 对齐 | 同列卡片左右边对齐；指标砖等分网格 |
| 主按钮 | 全页主按钮克制：横幅 CTA、上下文卡 CTA、待办「处理」 |
| Tag | 浅底彩字，小圆角 2px |
| 进度条 | 使用 Progress 线形；细高度约 4–8px；轨道中性、当前主色；见 progress-patterns |
| 图表 | 轴线浅灰，无立体 3D |
| 图标 | IconPark；宫格图标可多色，菜单/顶栏单色 currentColor |

## 5. 交付前检查（工作台）

- [ ] 壳仍为深蓝顶栏 + 白侧栏浅蓝选中（未抄参考图白顶栏/青绿主题）
- [ ] 左右栏比例合理，无大片空洞
- [ ] 有「概览数据」或「待办横幅」之一，避免只有空列表
- [ ] 指标砖/图表/表格密度接近 workbench-ops 气质
- [ ] 右栏有上下文或常用功能或日历（至少 2 块）
- [ ] IconPark 图标统一；无 Emoji
- [ ] 无界面规范说明文案
- [ ] 明暗切换时用 CSS 变量，暗色菜单选中不写死 `#E6F1FF`

## 7. 实现提示

- 布局：`Grid`/`Row`/`Col` + 多 `Card`
- 指标：`Statistic` 或自建指标砖；图表用业务已有图表库
- 列表：`List` / `Table size="small"`
- 步骤：组件库 `Steps` 或横幅内简易步骤
- 图标：IconPark（https://iconpark.oceanengine.com/official）
- 明暗：见 [theme-mode.md](theme-mode.md)

---

## 8. 反例

- 抄参考图的 **顶栏/品牌色/侧栏结构** 替换本规范壳
- 整页单一大卡无分区
- 只有装饰插画没有可操作入口
- 指标无单位、无对比、无业务名
- 表格无表头灰底、操作堆主按钮

## 工作台与页面标题

工作台**不需要**内容区「工作台」页头标题名；面包屑可选，直接进入待办/指标/列表等内容模块。
