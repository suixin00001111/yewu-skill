# Do / Don't（ui组业务组件）

## Do

- **顶栏通栏 `#004EA2` 白字**；侧栏白底；菜单选中 `#E6F1FF`
- 使用本规范组件库与官方 size / type / status
- 主按钮 `#165DFF`，语义色 success/warning/danger 分明
- 文字用四级灰色，不靠降低透明度硬造层级
- 列表页：筛选 + 表格 + 分页完整闭环（参考 table-list）
- 工作台：多卡片分区
- 详情页：标题+返回、蓝竖条分块、Descriptions；多阶段才加步骤条（参考 workbench），非营销首页
- 主按钮每个焦点区域仅一个
- 删除等危险操作二次确认；删除文字用红
- 空状态用 `Empty`，加载用 `Spin`/`Skeleton`
- 布局：深蓝顶栏 + 白侧栏 + 内容浅灰底 + 白卡片
- 表格操作列用 text 按钮，危险操作标红

## Don't

- **不要白顶栏 + 深色侧栏** 的反转壳（与确定样式相反）
- **不要**把顶栏改成渐变/玻璃/浅灰条
- **不要**菜单选中做成深色块或左侧粗竖条 Ant 默认风冒充
- 不要用 Ant Design 默认蓝 `#1677FF` 整套皮肤冒充本规范
- 不要用 Element Plus 默认绿/蓝体系
- 不要 AI 默认「紫渐变英雄区 + 大圆角玻璃卡片」
- 不要 16px+ 大圆角、夸张阴影、拟态
- 不要多主色按钮并排抢焦点
- 不要表格列无限堆且无固定操作列
- 不要表单一页上百字段不分 Card/Steps
- 不要用 Emoji 当图标系统（除非用户要求）
- 不要在代码里手写假组件却声称符合本规范
- 不要把营销落地页审美套到中后台密集表格页
- 不要详情页用大段纯文本堆字段（应用 Descriptions）
- 不要普通详情硬塞满屏 Steps

## 与其它体系的快速区分

| 体系 | 主色印象 | 特征 |
|------|----------|------|
| ui组业务组件（本规范） | 顶栏 `#004EA2` + 按钮 `#165DFF` | 白侧栏浅蓝选中、小圆角、四级灰 |
| Ant Design | `#1677FF` | 常见深色 sider 或另一套布局壳 |
| Semi Design | 另一套体系 | API/视觉不同，本 skill 默认不走 Semi |
| Element Plus | `#409EFF` | 偏 Vue 生态经典后台 |

若用户明确说 **Semi Design**，不要用本 skill 硬套；应换 Semi 规范或新建 skill。
