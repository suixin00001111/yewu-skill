# Do / Don't（ui组业务组件）

## Do

- 生成前写 Page Spec；交付前过 delivery-gates
- 列表/详情/工作台优先套 templates 再改业务
- 读操作有 loading/empty/error；写操作有 Message（async-states）
- 批量操作在表上工具行左侧；新建在右侧（ops-patterns）

- 弹窗：顶栏指定渐变+关闭；底栏右对齐次按钮+主色确认；确认类带语义图标；**文案按业务配置**（见 modal-patterns）

- 步骤条充满容器宽度，连接线均分；保留模块内边距

- 详情页头条：左标题 + 右返回/保存（有背景）

- **工作台不要**页面标题名

- 列表页简洁页面标题**单独一行**；新建等放表格上方工具行，**不与标题并列**

- 列表页标题单独一行；新建等主操作放表格上方工具行，不与标题并列

- 步骤条充满所在卡片/容器宽度，连接线均分拉伸；保留模块内边距

- 列表/普通业务页提供简洁页面标题（参考「产品列表」样式）；详情用标题+操作条；**工作台不要页头标题**

- 详情明细 Table 使用完整网格描边（`border` + `borderCell` 或等价全边框）

- 内容区与模块 Card 统一内边距 16–20px；Descriptions/分块表不贴卡片边

- **顶栏通栏 #004EA2 白字**；侧栏白底；菜单选中 #E6F1FF
- 使用本规范实现层组件与 size / type / status 枚举
- 主按钮 #165DFF，语义色 success/warning/danger 分明
- 文字用四级灰色，不靠透明度硬造层级
- **正文默认 14px / 行高约 22**；间距 4 的倍数；圆角 ≤8（见 style-usage / design-tokens）
- 列表页：筛选 + 表格 + 分页完整闭环
- 工作台：多卡片分区，非营销首页；左右栏与宫格见 content-ux
- 详情/表单：页面级保存/返回在最上方且页头有背景条；每模块单独一块；Steps 节点连线；Table 浅灰表头
- 主按钮每个焦点区域仅一个
- 进度：Progress 组件（当前主色 + 中性轨道）；业务阶段用 Steps
- 弹窗顶栏背景：`linear-gradient(180deg, #eaf0ff 0%, #ffffff 34%)`
- 反馈：轻结果 **Message**（删除成功/保存成功/失败等顶部提示，见 message-patterns）；删除确认 Popconfirm/Modal；复杂表单 Drawer/整页
- **删除成功必须 Message.success**，保存/提交成功同理；失败用 Message.error
- 空状态 Empty；加载 Spin/Skeleton
- 明暗：顶栏入口 + body 主题属性 + CSS 变量；顶栏两态 #004EA2
- 表格操作列 text 按钮；业务图标 IconPark outline

## Don't

- 不要跳过交付闸门直接交稿
- 不要静态假列表无任何加载/空态/接口占位（演示 mock 也需结构完整）
- 不要批量按钮只出现在行内、工具行无批量区

- 不要弹窗主按钮在左、无取消；不要弹窗无标题无关闭；不要长表单硬塞进过窄 Modal（改 Drawer）
- 不要把弹窗示例文案（邮寄设备/添加子账号/取消订单等）当成固定文案套进无关业务；标题正文按钮以用户要求为准


- 不要详情明细表只有横线、无网格描边


- 不要列表页标题与「新建」等按钮左右同一行并列


- 不要步骤条只挤在容器左侧一截、中间大片留白；不要为了拉满取消卡片内边距贴边


- 不要列表页只有面包屑没有页名；不要给工作台再加「工作台」标题条


- 不要在详情指标/明细表只画横线、无外框无竖线（与实现层有描边表不一致）


- 不要让 Descriptions/分块字段表 `padding:0` 贴满卡片边框（模块内容贴边）


- 不要用参考业务图替换本规范壳色/侧栏结构
- 不要工作台只有空列表而无指标/待办/右栏
- 不要假暗色（只改底色、组件仍浅色/硬编码 Light 色）
- 不要暗色下写死菜单选中 #E6F1FF 或顶栏非 #004EA2
- 不要白顶栏+深色侧栏反转壳；不要渐变/玻璃顶栏
- 不要其它库默认皮肤整站冒充本规范
- 不要 AI 紫渐变英雄区、大圆角玻璃卡、营销 Dashboard
- 不要多 primary 并排；不要 16px+ 圆角与重拟态阴影
- 不要用错反馈层级（删除不确认、成功却重型 Modal）
- 不要删除/保存成功后静默无提示；不要用成功 Modal 代替 Message
- 不要字段校验错误改用整页 Message 刷屏
- 不要表单百字段不分块；不要保存进表格操作列
- 不要箭头色块进度条；不要高饱和蓝表头
- 不要手绘假 Progress/假 Table；组件 API 跟实现层官方
- 不要把弹窗顶栏改成纯白或其它渐变（必须 `linear-gradient(180deg, #eaf0ff 0%, #ffffff 34%)`）
- 不要 Emoji/混图标库；不要手写假组件冒充规范
- **不要**在页面写规范注释、token 名、实现库品牌名

## 区分

| 体系 | 特征 |
|------|------|
| ui组业务组件 | 顶栏 #004EA2 + 按钮 #165DFF，白侧栏浅蓝选中，小圆角，产品态密度 |
| 其它后台默认壳 | 深色 sider / 不同主色等 — 用户未指定时不要硬套本 skill 去覆盖其它体系需求 |
