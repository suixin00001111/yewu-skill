# ui组业务组件 · 页面契约 Page Spec

生成可运行页或 UI 示意图前，**先写出本契约（可简写）**，再写代码。契约用于防漏模块、防错结构。

---

## 1. 必填字段

| 字段 | 说明 |
|------|------|
| 页面类型 | `workbench` / `list` / `detail` / `form-edit` / `settings` / 其它 |
| 业务名 | 系统名、页面名（列表标题用） |
| 路由/菜单 | 侧栏选中项、面包屑 |
| 模块列表 | 内容区有哪些 Card/区块（有序） |
| 页面级操作 | 返回/取消/保存/提交等（仅详情/表单顶条） |
| 主操作 | 列表「新建」等（工具行，不与标题并列） |
| 数据与接口 | 列表查询、详情加载、保存、删除的 **占位函数名** |
| 状态 | loading / empty / error 如何表现 |
| 反馈 | 成功/失败 Message 文案（可配置） |
| 弹窗/抽屉 | 是否有确认、表单弹窗、侧滑；文案来源用户 |
| 侧栏 | 默认可折叠 220↔48；Menu 与 Sider 同步（sider-collapse） |

## 2. 模板（复制填写）

```md
## Page Spec
- 类型：list
- 系统名：xxx管理系统
- 页面名：xxx列表
- 菜单：xxx管理 / xxx列表
- 模块：
  1. 筛选区（字段…）
  2. 工具行（新建…）
  3. Table + Pagination
- 页面级操作：无（列表）
- 接口占位：
  - fetchList(query) 
  - removeItem(id)
  - createItem(payload)
- 状态：首屏 Spin；无数据 Empty+新建；失败 Alert+重试
- 反馈：删除成功/失败 Message；保存成功 Message
- 弹窗：删除确认（标题/正文由业务）
```

## 3. 按页面类型的结构检查

| 类型 | 必须有 | 禁止 |
|------|--------|------|
| workbench | 待办/指标/列表等模块（见 content-ux） | 内容区「工作台」大标题条 |
| list | 页名单独一行；筛选；工具行；Table；分页 | 标题与新建并列 |
| detail / form-edit | 页头条（标题+返回/保存）；分模块 Card | 页面保存塞进表格操作列 |
| 含步骤 | Steps 满宽+模块内边距 | 箭头色块假进度条 |

## 4. 与代码模板

契约写完后，套用 `references/templates/` 下对应骨架，替换业务字段与接口占位。
