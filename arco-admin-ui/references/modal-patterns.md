# ui组业务组件 · 弹窗 Modal 样式与用法

> 对照业务弹窗参考图：确认提示、表单弹窗、带输入的确认弹窗。  
> 实现用真实 `Modal`（及 Form / Input / Select）；界面不出现第三方库品牌名。  
> 主色按钮 `#165DFF`；圆角约 8px；遮罩浅灰半透明。

---

## 文案规则（强制）

弹窗**结构/样式**固定（顶栏渐变、关闭、底栏按钮顺序、三类布局），**文案全部按业务与用户要求配置**，本文件中的标题、正文、按钮字仅为**结构示例**，禁止当死文案套用。

| 可配置项 | 说明 |
|----------|------|
| 标题 | 由业务决定，如「提示」「删除确认」「新建xxx」；用户指定则用用户文案 |
| 正文 / 问句 | 按场景编写；可带业务对象名、数量等动态字段 |
| 表单 label / placeholder / helper | 按字段字典与用户要求 |
| 底栏按钮 | 常见「取消 + 确认/确定/确认提交」，**文案以用户/产品为准**（可「关闭」「我知道了」「提交」等） |
| 图标语义 | 仍按 info/warning/danger/success 选色；是否显示图标可按场景 |
| 宽度 | 确认类/表单类区间内按内容调整 |

### 生成时

1. 优先采用**用户提示中的原文案**（标题、按钮、说明句）。  
2. 用户未给文案时：用简短中性业务话术，并与操作动词一致（删→删除确认/删除成功）。  
3. **不要**把示例「邮寄设备」「添加子账号」「取消订单」写进无关业务页。  
4. 界面仍禁止规范注释、token 名、外部品牌名。

## 0. 参考图对照（生成时必须贴合）

| 结构类型 | 标题（示例，可改） | 正文结构（文案可改） | 底栏（文案可改） |
|----------|----------|----------|------|
| 确认提示 | 提示 | 主色信息图标 + 一句确认问句（如「确认已经将设备邮寄给客户？」） | 取消 · 确认 |
| 表单弹窗 | 添加子账号 等业务名 | 垂直表单项（手机号/密码/确认密码/角色…）；密码可显隐；字段下灰字规则 | 取消 · 确认提交 |
| 确认+输入 | 取消订单 等动作名 | 警告图标 + 说明；金额输入 + 后缀「元」；下方不可撤销灰字提示 | 取消 · 确定 |

样式：顶栏背景统一 `linear-gradient(180deg, #eaf0ff 0%, #ffffff 34%)`，右上角 **×** 关闭；主色按钮统一 `#165DFF`。

## 1. 通用壳（三种共用）

| 元素 | 规范 |
|------|------|
| 容器 | 白底卡片，圆角 **8px**，中等阴影；水平居中，垂直略偏上或居中 |
| 遮罩 | 半透明浅灰/黑（约 40–50%），点击遮罩默认可关（危险操作可禁） |
| 顶栏 | **背景** `linear-gradient(180deg, #eaf0ff 0%, #ffffff 34%)`；左标题 + 右关闭 ×；底部分割可无或极淡 |
| 标题 | 左：标题 16px / 字重 500–600 / `color-text-1`；右：**关闭 ×**（灰，hover 加深） |
| 内容区 | 内边距约 **20–24px**；正文 14px；上下节奏 16 |
| 底栏 | 右对齐按钮组；**次按钮在左、主按钮在右**；间距 8–12 |
| 按钮 | 取消 = default 描边/浅底；确认/提交 = `type="primary"`（`#165DFF`） |
| 宽度 | 确认类约 **400–480**；表单类约 **480–560**；勿全屏除非业务要求 |



### 顶栏背景（强制 CSS）

```css
.modal-header /* 或实现层 Modal 标题区自定义 */ {
  background: linear-gradient(180deg, #eaf0ff 0%, #ffffff 34%);
}
```

实现层可用 `Modal` 的 `className` / 包裹标题区样式覆盖；**不要**改成白顶栏或其它渐变。

### 底栏按钮顺序（强制）

```
[取消]  [确认 / 确认提交 / 确定]
```

- 同一底栏 **仅一个** primary。  
- 危险确认仍可用 primary；文案用「确定/确认」，删除类可 `status=danger`（按业务）。

## 2. 类型 A · 纯确认提示（如图「提示」）

**场景**：二次确认、无额外字段（邮寄确认、删除确认等）。

| 区域 | 内容 |
|------|------|
| 标题 | **可配置**；缺省可用「提示」或动作名，用户指定则从其文案 |
| 正文 | **信息图标**（按语义）+ **用户/业务确认问句**（勿写死示例句） |
| 图标 | 信息 = 主色蓝；成功绿 / 警告橙 / 错误红 按语义切换 |
| 底栏 | 次按钮 + 主按钮；默认「取消」「确认」，**文案可配置** |

```tsx
// 以下为结构示例：title/content/okText/cancelText 均由业务传入
<Modal
  title={title} // 例：提示
  visible={visible}
  onCancel={onCancel}
  onOk={onOk}
  okText="确认"
  cancelText="取消"
>
  <div style={{ display: 'flex', gap: 8, alignItems: 'flex-start' }}>
    {/* IconPark 信息图标，主色 */}
    <span>确认已经将设备邮寄给客户？</span>
  </div>
</Modal>
```

**不要**：无标题只有正文；主按钮在左；缺少取消。

## 3. 类型 B · 表单弹窗（如图「添加子账号」）

**场景**：新建/编辑短表（字段 roughly ≤ 8）；过长表单改用 Drawer 或整页。

| 区域 | 内容 |
|------|------|
| 标题 | **业务名/用户指定** + 关闭 |
| 表体 | `Form` **vertical**；标签在上；必填 **红色 \*** |
| 控件 | Input / Password（可显隐）/ Select 等，宽度 **100%** |
| 说明 | 字段下 helper：12–13px、`color-text-3`（如密码规则） |
| 底栏 | 次 + 主（默认「取消」「确认提交」）；**按钮文案可配置** |

### 表单项节奏

- 项间距约 **16–20px**
- placeholder：「请输入… / 请选择…」
- 校验错误贴字段下方，红字

```tsx
<Modal title="添加子账号" visible={visible} onCancel={...} onOk={...} okText="确认提交" cancelText="取消">
  <Form layout="vertical" ...>
    <Form.Item label="手机号" field="mobile" rules={[{ required: true }]}>
      <Input placeholder="请输入手机号" />
    </Form.Item>
    <Form.Item label="密码" field="password" rules={[{ required: true }]} extra="输入6-16位数字、字母的组合">
      <Input.Password placeholder="请输入密码" />
    </Form.Item>
    ...
  </Form>
</Modal>
```

**不要**：标签与输入横排挤在窄弹窗；必填无红星；底栏只有确认没有取消。

## 4. 类型 C · 确认 + 输入（如图「取消订单」）

**场景**：确认同时采集一项关键数据（退款金额、原因等）。

| 区域 | 内容 |
|------|------|
| 标题 | **动作名/用户指定** |
| 提示行 | **警告/信息图标**（警告用橙）+ 说明文案 |
| 输入 | 单行 Input；可 **后缀单位**（如「元」）用 `Input` + `addAfter` / 组合框 |
| 辅助说明 | 输入下方灰字风险提示（不可撤销等） |
| 底栏 | 次 + 主（默认「取消」「确定」）；**文案可配置** |

```tsx
<Modal title="取消订单" visible={visible} onCancel={...} onOk={...} okText="确定" cancelText="取消">
  <div style={{ display: 'flex', gap: 8, marginBottom: 12 }}>
    {/* 警告图标 橙色 */}
    <span>确定取消订单，请输入退款金额</span>
  </div>
  <Input placeholder="请输入金额" addAfter="元" />
  <div style={{ marginTop: 8, color: 'var(--color-text-3)', fontSize: 13 }}>
    取消订单后，会退款对应的金额，此操作无法撤回！
  </div>
</Modal>
```

## 5. 语义图标色

| 语义 | 色 | 用途 |
|------|-----|------|
| 信息 | `#165DFF` / primary | 一般确认、说明 |
| 警告 | `#FF7D00` | 撤销、退款、风险操作说明 |
| 危险 | `#F53F3F` | 删除、不可恢复（可配合危险按钮） |
| 成功 | `#00B42A` | 完成类提示（少用于确认前） |

图标统一 **IconPark** outline/实心小圆标，与正文对齐。

## 6. 与 Drawer / Message 的分工

| 组件 | 何时用 |
|------|--------|
| **Modal 确认 A** | 短确认、无字段或极少字段 |
| **Modal 表单 B** | 短表新建/编辑 |
| **Modal 确认+输入 C** | 确认且必须填 1 个关键值 |
| **Drawer** | 字段多、需对照页内上下文 |
| **Message** | 已成功/失败的轻反馈，不再弹确认框重复；**删除/保存完成后必须 Message**，见 [message-patterns.md](message-patterns.md) |

删除：优先 `Popconfirm` 或 Modal A 做确认；确认成功后 **必须** `Message.success`，失败 `Message.error`；勿只 Message 不确认，也勿确认后静默。
