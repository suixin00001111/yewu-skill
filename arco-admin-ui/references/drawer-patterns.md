# ui组业务组件 · 抽屉 Drawer

与 Modal 分工：字段多、需对照列表上下文、详情侧滑时用 Drawer。

---

## 1. 何时用

| 用 Drawer | 用 Modal |
|-----------|----------|
| 字段 > 8 或分区较多 | 短确认、短表（≤8 字段） |
| 从列表点「查看/编辑」不离页 | 强打断确认 |
| 需要较大表单宽度 | 窄确认框 |

## 2. 结构

| 区域 | 规范 |
|------|------|
| 宽度 | 常用 480 / 560 / 640；详情可 720；勿默认全屏 |
| 标题 | 左：业务标题（**可配置**）；右：关闭 |
| 内容 | 内边距 16–20；分模块可用锚点/小标题 |
| 底栏 | **右对齐**：取消/关闭 + 主操作（保存/提交）；文案可配置 |
| 遮罩 | 可点遮罩关闭（有未保存表单时建议拦截） |

## 3. 与页面级保存

- 列表上的侧滑编辑：保存放在 **Drawer 底栏**  
- 整页详情编辑：仍用 **页头条** 保存，不用 Drawer 重复一套全页保存  

## 4. 反馈

保存成功：`Message.success` + 关闭 Drawer + 刷新列表。  
文案可配置。

## 5. 自检

- [ ] 底栏右对齐，主按钮一个  
- [ ] 宽度足够，不挤成长 Modal  
- [ ] 文案来自业务
