# ui组业务组件 · 布局模式

默认壳以 [product-shell.md](product-shell.md) 为准。下方描述与参考截图一致。

## 1. 默认产品壳（必须，优先）

```
┌────────────────────────────────────────────────┐
│  Header 通栏 #004EA2  系统名(业务)   工具/头像   │  ~52px
├──────────┬─────────────────────────────────────┤
│ 白侧栏   │ Breadcrumb                          │
│ Menu     │                                     │
│ 选中     │  白卡片业务区                        │
│ #E6F1FF  │  （筛选 / 表格 / 工作台 / 详情…）     │
│          │                                     │
└──────────┴─────────────────────────────────────┘
```

- **Header**：通栏深蓝 `#004EA2`，白字；整行通栏
- **Sider**：**默认可折叠**；白底，展开 220px / 收起 48px 仅图标；Menu 选中 `#E6F1FF` 且 `collapse` 同步；**必读** [sider-collapse.md](sider-collapse.md)
- **Content** 背景：`#F2F3F5` / `#F7F8FA`；内容卡片白底；内边距 16–24px

参考图：`examples/workbench.png`、`table-list.png`、`detail-with-steps.png`、`detail-plain.png`

## 2. 顶栏主导（仅用户明确要求时）

默认 **顶栏 + 侧栏**，不要默认成无侧栏。

## 3. 列表页结构（参考 table-list.png）

1. Breadcrumb  
2. 可选分段按钮/Tab  
3. 筛选区：查询 +「查询」主按钮 +「重置」  
4. 工具栏：右侧「+ 添加」等  
5. Table（可树形）  
6. Pagination  

操作列：蓝字链接 + 红色删除。

## 4. 工作台结构（参考 workbench.png / workbench-ops.png）

- 可选顶工具条：业务 Tab + 搜索  
- 左主栏：待办/告警横幅（可含步骤）→ 指标砖+趋势图 → 业务动态/巡检 → 内嵌 Tab 表格  
- 右辅栏：上下文对象卡 → 常用功能宫格 → 变更日历/日程  
- 亦可组合传统：快捷操作、任务列表、通知、用户卡（按业务裁剪）  
- 多卡片分区，不要大英雄区  
- **内容区质感细则**见 [content-ux.md](content-ux.md)（**不改**顶栏/菜单色；参考图主题色映射为本规范蓝灰）

## 5. 详情页结构（参考 detail-with-steps / detail-plain）

1. 页头：页面级操作栏 — 标题（左）+ 返回/保存/提交（右）；**白/浅灰背景容器**；全部在最上方  
2. **可选** `Steps` 步骤条（多阶段才加；节点+连接线，非箭头色块；当前 process / 未完成 wait）  
3. 每一模块单独一块（独立 Card）：蓝竖条标题 + Descriptions/子表  
4. 分块内可嵌 Table（浅灰表头、组件库默认表格气质）  
5. 普通详情默认 **无步骤条**；含流程/阶段/向导语义时用 **有步骤条**

详见 [product-shell.md](product-shell.md) B3。

## 6. 表单页结构

- **页面级操作（保存/返回/取消）固定顶部页头**，不进表格、不按模块拆多套保存
- **每个业务模块单独一块** Card

- 单栏宽表单：最大宽 640–800  
- 多区块：多个 Card 分组  
- 底部操作栏可 Affix，主次分明

## 7. 仪表盘

- 首行 4 个 Statistic；次行图表 + 列表  
- 卡片圆角 4–8，留白一致

## 8. 断点（参考）

| 名称 | 宽度 | 行为 |
|------|------|------|
| xs | <576 | 侧栏改抽屉 |
| sm | ≥576 | 单列堆叠筛选 |
| md | ≥768 | 筛选两列 |
| lg | ≥992 | 标准侧栏 |
| xl | ≥1200 | 宽松边距 |
| xxl | ≥1600 | 内容最大宽可限制 |

桌面 mockup 默认 **1440** 或 **1920×1080**。

## 9. 层级与表面

- 顶栏实色 `#004EA2`；页面底浅灰；内容白 Card  
- 弹层 Modal/Drawer 最上层

## 10. 导航信息架构

- 一级业务域 + 二级资源；Menu、Breadcrumb 与当前页一致


## Steps 步骤条宽度（强制）

- 步骤条必须 **充满其所在白色容器/模块内容区的宽度**（`width: 100%`）。
- **保留容器内边距**：由外层 Card / page 模块的 16–20px 内边距承担，步骤条不要再缩成左对齐一小截。
- 布局：节点 `flex: none`；**连接线 `flex: 1`** 均分拉满中间空隙；步骤之间视觉间距靠 line 的水平 margin（约 8–12px）。
- 实现层：`Steps` 放在全宽容器内；勿给 Steps 固定过窄 max-width；水平 `direction=horizontal` 铺满。
- 禁止：步骤条挤在容器左侧、中间大片空白；禁止为了拉满而取消卡片内边距贴边。

## 页面标题与页头操作（canonical）

### 按页面类型

| 页面类型 | 页面标题 | 主操作位置 |
|----------|----------|------------|
| **工作台** | **不要**单独页头标题名（不要「工作台」大标题条） | 内容区内业务按钮；无整页标题行 |
| **列表 / 普通业务页** | **需要**简洁页名（参考「产品列表」：16px / 字重 500，左对齐） | 标题**单独一行**；「新建」等放在**筛选下方、表格上方**工具行右对齐，**禁止与标题左右并列** |
| **详情 / 表单编辑** | **需要** | 页头条（有背景容器）：左标题 + 右返回/取消/保存；页面级操作仅在此条 |

### 列表页推荐结构

```
面包屑（可选）
┌─ 白卡片 ─────────────────────────────┐
│  页面标题（单独一行，无右侧并列按钮）   │
│  筛选区 + 查询 / 重置                  │
│  表格工具行（右：新建等）               │
│  Table + Pagination                    │
└────────────────────────────────────────┘
```

### 禁止

- 工作台再套「工作台」页头标题
- 列表页只有面包屑、没有页面名称
- 列表页标题与「新建」等主按钮同一行左右并列
- 把列表标题做成营销大标题 / 厚重双行英雄区


## Table 描边与尺寸（对齐实现层）

| 场景 | 要求 |
|------|------|
| 详情分块明细表 | 必须 `border` + `borderCell`（完整网格） |
| 列表主表 | 至少 `border`；需要列网格时加 `borderCell` |
| 表头 | 浅灰；禁止高饱和蓝表头 |
| padding | 默认约 **9px 16px**；`size="small"` 约 **5px 16px** |
| 边框色 | 中性，禁止主色描边表 |
