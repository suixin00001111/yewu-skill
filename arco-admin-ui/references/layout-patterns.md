# ui组业务组件 · 布局模式

默认壳以 [product-shell.md](product-shell.md) 为准。下方描述与参考截图一致。

## 1. 默认产品壳（必须，优先）

```
┌────────────────────────────────────────────────┐
│  Header 通栏 #004EA2  系统名(业务)   工具/头像   │  ~52px
├──────────┬─────────────────────────────────────┤
│ 白侧栏   │ Breadcrumb                          │
│ Menu     │                                     │
│ 选中     │  白卡片业务区                        │
│ #E6F1FF  │  （筛选 / 表格 / 工作台 / 详情…）     │
│          │                                     │
└──────────┴─────────────────────────────────────┘
```

- **Header**：通栏深蓝 `#004EA2`，白字；整行通栏
- **Sider**：白底，宽展开 ~200–220px；Menu 选中 `#E6F1FF`
- **Content** 背景：`#F2F3F5` / `#F7F8FA`；内容卡片白底；内边距 16–24px

参考图：`examples/workbench.png`、`table-list.png`、`detail-with-steps.png`、`detail-plain.png`

## 2. 顶栏主导（仅用户明确要求时）

默认 **顶栏 + 侧栏**，不要默认成无侧栏。

## 3. 列表页结构（参考 table-list.png）

1. Breadcrumb  
2. 可选分段按钮/Tab  
3. 筛选区：查询 +「查询」主按钮 +「重置」  
4. 工具栏：右侧「+ 添加」等  
5. Table（可树形）  
6. Pagination  

操作列：蓝字链接 + 红色删除。

## 4. 工作台结构（参考 workbench.png）

- 左主栏：快捷操作 → 任务/待办 → 常用系统/通知  
- 右辅栏：用户卡、日程  
- 多卡片分区，不要大英雄区

## 5. 详情页结构（参考 detail-with-steps / detail-plain）

1. 页头：标题（左）+「返回」（右）  
2. **可选** 横向流程步骤条（多阶段才加；当前步主蓝，未完成灰）  
3. 分块：左侧蓝竖条标题 + `Descriptions`（border、标签浅灰底、多列）  
4. 分块内可嵌 Table  
5. 普通详情默认 **无步骤条**；含流程/阶段/向导语义时用 **有步骤条**

详见 [product-shell.md](product-shell.md) B3。

## 6. 表单页结构

- 单栏宽表单：最大宽 640–800  
- 多区块：多个 Card 分组  
- 底部操作栏可 Affix，主次分明

## 7. 仪表盘

- 首行 4 个 Statistic；次行图表 + 列表  
- 卡片圆角 4–8，留白一致

## 8. 断点（参考）

| 名称 | 宽度 | 行为 |
|------|------|------|
| xs | <576 | 侧栏改抽屉 |
| sm | ≥576 | 单列堆叠筛选 |
| md | ≥768 | 筛选两列 |
| lg | ≥992 | 标准侧栏 |
| xl | ≥1200 | 宽松边距 |
| xxl | ≥1600 | 内容最大宽可限制 |

桌面 mockup 默认 **1440** 或 **1920×1080**。

## 9. 层级与表面

- 顶栏实色 `#004EA2`；页面底浅灰；内容白 Card  
- 弹层 Modal/Drawer 最上层

## 10. 导航信息架构

- 一级业务域 + 二级资源；Menu、Breadcrumb 与当前页一致
