# ui组业务组件 · 组件视觉规格（component-visual）

> 对齐实现层组件库**既有**组件视觉与尺寸，保证生成页「像真实组件」而非手绘。  
> 对外只称 **ui组业务组件**；代码用真实组件 API。  
> 产品壳色仍以 product-shell 为准（顶栏 `#004EA2` 等）。

---

## 1. Avatar 头像

### 1.1 形态

| 属性 | 规范 |
|------|------|
| 形状 `shape` | 默认 **`circle`**；列表/卡片可用 `square`（圆角 medium ≈ 4px） |
| 内容类型 | 图片 / 文字（姓或名 1–2 字）/ 图标 |
| 默认尺寸 | **40×40**（未传 size 时） |
| 字号 | 默认约 **size/2**（40 → 20）；`autoFixFontSize` 默认开启，文字过长自动缩放 |
| 背景 | 无图时用填充色 `var(--color-fill-4)` 一类中性填充 |
| 文字色 | 白 / 高对比浅色；字重约 500 |
| 盒模型 | inline-flex 居中；`box-sizing: border-box` |

### 1.2 推荐尺寸（中后台）

| 场景 | size (px) | 说明 |
|------|-----------|------|
| 顶栏用户 | **28–32** | 与顶栏高度 52 协调，勿大于 36 |
| 表格行 / 评论 | **24–32** | 紧凑 |
| 表单/资料卡默认 | **40** | 组件默认 |
| 个人中心/详情强调 | **48–64** | 少用更大 |

也可用实现层 `size` 数字；与按钮 `size` 枚举不是同一套时以 **px 数字** 为准。

### 1.3 头像组 Avatar.Group

| 规则 | 说明 |
|------|------|
| 叠放 | 后续头像负边距约 **-10px** 叠在前一项上 |
| 描边 | 组内头像 **2px** 描边，色为页面/卡片底 `var(--color-bg-2)`，避免糊成一团 |
| 超量 | `maxCount`；多余显示 **`+N`**，可气泡展开 |
| z-index | 默认递减；需要时可 `zIndexAscend` |

### 1.4 交互头像

| triggerType | 表现 |
|-------------|------|
| `button`（默认） | 右下角约 **20×20** 圆形操作钮（上传/编辑） |
| `mask` | hover 整面半透明遮罩 + 居中图标 |

中后台顶栏一般 **不需要** trigger；资料编辑页可用。

### 1.5 用法要点

```tsx
import { Avatar } from '@/components'; // 实现层组件库入口（以项目为准）

// 顶栏
<Avatar size={32} style={{ background: 'rgba(255,255,255,.28)' }}>张</Avatar>

// 表格
<Avatar size={28}>李</Avatar>

// 图片
<Avatar size={40}><img alt="用户" src={url} /></Avatar>

// 组
<Avatar.Group size={32} maxCount={3}>
  <Avatar>A</Avatar><Avatar>B</Avatar><Avatar>C</Avatar><Avatar>D</Avatar>
</Avatar.Group>
```

- 顶栏深色底上：半透明浅底 + 白字，或浅色图；**不要**用深灰块看不见。
- 禁止用 Emoji 当头像体系；图标型头像可用 IconPark 缩小居中。
- 与姓名并排时：头像 + 8px 间距 + 姓名（14px）。

## 2. 其它常用组件视觉（与实现层一致）

### 2.1 Tag 标签

- 高度约 **22–24**；字号 **12**；圆角 **2**（small）
- 状态：色 + **文字**（进行中/已完成/预警）；浅底 + 语义字色
- 勿做成大胶囊营销标签

### 2.2 Badge 徽标

- 圆点或数字角标贴在头像/图标右上
- 数字封顶展示（如 99+）；危险用 danger 色

### 2.3 Button（尺寸回顾）

| size | 高度约 |
|------|--------|
| mini | 24 |
| small | 28 |
| default | 32 |
| large | 36–40 |

主色 `#165DFF`；同一焦点区仅一个 primary。

### 2.4 Table

## Table 描边（对齐实现层）

实现层 `Table` 默认多为**行间横线**；需要完整网格时必须显式：

| API | 效果 |
|-----|------|
| 默认 | 表头浅底 + 行底部分割线（无竖线、可无外框） |
| `border` | **外边框** + 首列左边线等容器描边 |
| `borderCell` | **单元格竖线/网格**（与横线一起形成完整格线） |

### 场景约定（强制）

| 场景 | 用法 |
|------|------|
| 详情分块内明细表（如指标明细） | **`border` + `borderCell`**（或等价：th/td 全边框 `1px solid` 边框色 ≈ `#E5E6EB` / `var(--color-neutral-3)` / `var(--color-border-2)`） |
| Descriptions 字段表 | 本身带单元格边框，保持网格 |
| 列表页主表 | 推荐 `border`；信息密集可再加 `borderCell`；至少不要「完全无任何分割」 |
| 工作台内嵌简表 | 至少行分割；需要结构化对比时用 `border` + `borderCell` |

### 视觉

- 表头：浅灰 `fill-1` / `bg-2`，**不要**高饱和蓝表头
- 边框色：中性边框，勿用主色描边表格
- 单元格 padding 约 9–12px；字号 14
- 禁止只画横线却在规范里写「有描边表」却交付无竖线网格（详情明细场景）

```tsx
<Table
  border
  borderCell
  size="small"
  columns={columns}
  data={data}
  pagination={false}
/>
```


### 2.5 Descriptions

- 标签浅底、值白底；单元格 padding 约 10–12
- **外包在 Card 内边距 16–20 内**，不贴卡片边

### 2.6 Steps

- 节点圆 + 连接线；当前主色实心；完成主色描边/勾；等待灰

### 2.7 Card / 模块

- 圆角 4–8；边框 border-2；内边距 16–20；块间距 16

### 2.8 Message / Modal

- **Message（全局提示）**：顶部居中；白底/浮层 + 浅描边 + 轻阴影；左类型图标（success 绿 / error 红 / warning 橙 / info 主色蓝）+ 右 14px 文案；默认约 3s 自动关闭；多条纵向堆叠间距约 16
- 删除完成、保存成功等结果反馈**必须** Message，细则见 [message-patterns.md](message-patterns.md)
- Modal：顶栏 `linear-gradient(180deg, #eaf0ff 0%, #ffffff 34%)` + 中层阴影 + 右对齐取消/主色确认；三类见 modal-patterns；**不用于**单纯「操作成功」结果
- 确认删除用 Popconfirm / Modal 类型 A → 成功后再 Message



### 2.9 Progress 进度条

- 线形：轨道 `color-fill-3` 中性；内条 primary / status 色；胶囊圆角；文案 12px 在右
- 状态：normal / success / warning / error
- 与 Steps 区分；细则 [progress-patterns.md](progress-patterns.md)

## 3. 生成时自检（组件像不像库）

- [ ] Avatar 默认圆、尺寸合理，非常规大圆或方形生硬块
- [ ] 文字头像自动缩放、不溢出
- [ ] 头像组有叠压与描边
- [ ] Tag/按钮/表格密度与 size 阶梯一致
- [ ] 未手写一套完全偏离组件库的控件皮肤

## Steps 步骤条宽度（强制）

- 步骤条必须 **充满其所在白色容器/模块内容区的宽度**（`width: 100%`）。
- **保留容器内边距**：由外层 Card / page 模块的 16–20px 内边距承担，步骤条不要再缩成左对齐一小截。
- 布局：节点 `flex: none`；**连接线 `flex: 1`** 均分拉满中间空隙；步骤之间视觉间距靠 line 的水平 margin（约 8–12px）。
- 实现层：`Steps` 放在全宽容器内；勿给 Steps 固定过窄 max-width；水平 `direction=horizontal` 铺满。
- 禁止：步骤条挤在容器左侧、中间大片空白；禁止为了拉满而取消卡片内边距贴边。

## 页面标题与页头操作（canonical）

### 按页面类型

| 页面类型 | 页面标题 | 主操作位置 |
|----------|----------|------------|
| **工作台** | **不要**单独页头标题名（不要「工作台」大标题条） | 内容区内业务按钮；无整页标题行 |
| **列表 / 普通业务页** | **需要**简洁页名（参考「产品列表」：16px / 字重 500，左对齐） | 标题**单独一行**；「新建」等放在**筛选下方、表格上方**工具行右对齐，**禁止与标题左右并列** |
| **详情 / 表单编辑** | **需要** | 页头条（有背景容器）：左标题 + 右返回/取消/保存；页面级操作仅在此条 |

### 列表页推荐结构

```
面包屑（可选）
┌─ 白卡片 ─────────────────────────────┐
│  页面标题（单独一行，无右侧并列按钮）   │
│  筛选区 + 查询 / 重置                  │
│  表格工具行（右：新建等）               │
│  Table + Pagination                    │
└────────────────────────────────────────┘
```

### 禁止

- 工作台再套「工作台」页头标题
- 列表页只有面包屑、没有页面名称
- 列表页标题与「新建」等主按钮同一行左右并列
- 把列表标题做成营销大标题 / 厚重双行英雄区

## Modal 弹窗视觉（摘要）

| 项 | 规范 |
|----|------|
| 容器 | 白底；圆角 **8px**；中层阴影；水平居中 |
| 顶栏 | `linear-gradient(180deg, #eaf0ff 0%, #ffffff 34%)`；左标题 16/500–600；右关闭灰字 |
| 内容 | 内边距 20–24；正文 14；确认类带语义图标（信息 `#165DFF` / 警告 `#FF7D00`） |
| 底栏 | 右对齐；取消 default 在左；主操作 primary `#165DFF` 在右 |
| 表单 | vertical；标签上置；必填红星；控件 width 100%；helper 12–13 / text-3 |
| 单位 | 金额等用 Input 后缀/addAfter（如「元」） |

三类模板与代码骨架见 [modal-patterns.md](modal-patterns.md)。**标题/正文/按钮文案由业务配置，示例非固定。**
