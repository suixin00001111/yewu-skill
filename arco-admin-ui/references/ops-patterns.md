# ui组业务组件 · 批量操作 / 权限 / 空态补充

---

## 1. 批量操作（列表）

| 项 | 规范 |
|----|------|
| 选择 | Table `rowSelection`；跨页策略按业务（默认同页） |
| 工具位 | 表格上方工具行**左侧**：已选 N 项 + 批量删除/导出等；**右侧**仍放新建 |
| 无选中 | 批量按钮 `disabled` 或点击 Message.warning（文案可配置） |
| 批量删除 | 确认 Modal/Popconfirm → 成功 Message → 刷新并清空选择 |
| 导出 | 按钮 loading；完成 Message.success |

不要把批量按钮塞进每一行操作列。

## 2. 权限与按钮

| 场景 | UI |
|------|-----|
| 无权限操作 | 按钮不渲染，或 disabled + Tooltip「无权限」 |
| 无菜单权限 | 侧栏不展示；直链进入可 403 |
| 403 页 | Result 类布局 + 返回首页/上一页（文案可配置） |

代码可预留：

```ts
// 以项目权限工具为准
const can = (code: string) => auth.codes.includes(code);
{can('xxx:create') && <Button type="primary">新建</Button>}
```

不要在界面写「权限码：xxx」调试字样（除非用户要）。

## 3. 空态 Empty

| 场景 | 建议 |
|------|------|
| 列表无数据 | 插画/图标 + 说明 + 主按钮「新建」（若有权限） |
| 筛选无结果 | 说明「无匹配数据」+「重置筛选」 |
| 详情缺失 | Result/Empty + 返回列表 |

文案可配置；主按钮最多一个 primary。

## 4. 上传/附件（轻量）

- 详情「附件」模块单独 Card  
- 上传中可用 Progress 或按钮 loading  
- 列表可用「附件数」列；预览/下载为 text 按钮  
- 失败 Message.error  

## 5. 筛选区密度

- 默认一行主筛选 + 查询/重置；字段多时「展开/收起」高级筛选  
- 查询 primary，重置 default  
- 与 Table 间距 16  

## 6. 自检

- [ ] 批量在工具行左，新建在右  
- [ ] 危险批量有确认 + Message  
- [ ] 空态可行动  
- [ ] 权限不在 UI 暴露内部码（除非需要）
