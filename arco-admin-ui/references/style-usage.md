# ui组业务组件 · 样式与组件用法（style-usage）

> 本文件把 **样式规范 + 组件用法** 收敛为可执行规则，用于生成/评审中后台界面。  
> 对外与界面文案只称 **ui组业务组件**；实现层用真实组件库 API 保证可运行。  
> **不改** 产品壳：顶栏 #004EA2、白侧栏、选中 #E6F1FF（见 product-shell）。

---

## 1. 设计原则（产品态）

| 原则 | 落地要求 |
|------|----------|
| **清晰** | 信息层级靠字号/字重/四级灰，不靠装饰；字段有标签，状态有文案 |
| **高效** | 列表可扫读；主操作唯一且靠前；筛选→查询→结果闭环 |
| **一致** | 同场景同组件、同尺寸阶梯、同间距节奏；按钮动词全流程一致 |
| **包容** | 对比度足够；图标按钮有 title/aria-label；状态不只靠颜色（Tag 带文字） |

始终处于 **产品态（中后台）**：优先密度与可扫描，禁止营销落地页构图。

## 2. 样式系统速查（与 design-tokens 一致）

### 2.1 色
- **功能色分工**：主色行动（查询/新建/提交）；成功完成；警告风险；危险删除/失败；信息用中性或主色浅底。
- **四级文字**：text-1 标题/强调；text-2 正文；text-3 辅助/占位；text-4 禁用。
- **表面**：页面 fill-2 浅灰底 → 白 Card → 表头/描述标签用 fill-1/bg-2。
- **边框**：默认 border-2；勿用高饱和色当分割线。
- **链接/行操作**：与主色一致；危险操作用 danger 文案色。

### 2.2 字体与段落
| 角色 | 字号 | 行高 | 字重 |
|------|------|------|------|
| 展示/少用 | 24 / 20 | 32 / 28 | 500–600 |
| 区块标题 | 16 | 24 | 500 |
| 正文/表格/表单 | **14** | **22** | 400 |
| 辅助/说明 | 12–13 | 18–20 | 400 |

- 默认正文 **14px**；段落行高约 **1.5× 字号**（14 → 约 21–22）。
- 按钮用动词「保存/提交/查询」；空态说明原因+下一步；错误说明问题与修法。

### 2.3 间距（4 的倍数）
4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48

| 场景 | 取值 |
|------|------|
| 控件间隙 | 8–12（Space） |
| 表单项纵向 | 16–20 |
| 卡片内边距 | 16–20 / 24 |
| 模块间距 | 16–20 |
| 内容区内边距 | 16–24 |
| 页头条与下方 | 16 |

禁止随意奇数间距；禁止落地页式超大留白当默认。

## 模块与内容区内边距（统一，强制）

- **内容区页边距**：相对主内容容器统一 **16–20px**（桌面可 20–24），左右上下一致，禁止一侧贴边。
- **模块 Card**：块间距 **16px**；卡片 **内容区内边距统一 16–20px**（标题区与体可略有分割线，但左右边距一致）。
- **Descriptions / 描述表 / 分块字段表**：必须落在模块内边距之内，**禁止** `padding:0` 贴满卡片四边导致标签/值顶住边框。
- **嵌套 Table**：表可占满内容区宽度，但表外仍保留模块内边距；不要让表格贴住页面灰底。
- **页头操作条**：同样左右与内容区对齐（同一套页边距），内边距 12–16。


### 2.4 圆角与阴影
| 级别 | 圆角 | 用途 |
|------|------|------|
| 小 | 2px | Tag |
| 中 | **4px** | 按钮、输入、菜单选中 |
| 大 | 8px | Card、Modal |
| 无 | 0 | 贴边表格 |

卡片低海拔或仅边框；弹层中海拔。禁止 16px+ 大圆角与彩色重阴影。

### 2.5 尺寸阶梯
mini | small | default | large  
表格/筛选优先 small 或 default；同排 size 一致。

### 2.6 图标尺寸
表单/按钮 14–16；顶栏/侧栏 16–18；工作台宫格 20–22。业务图标 **IconPark outline** + currentColor。

### 2.7 栅格与对齐
Grid/Row/Col，gutter 16–20；label 对齐全页统一；数字列右对齐；操作列右固定。

## 3. 组件用法（按场景）

### 3.1 布局壳
Layout + Header/Sider/Content；Menu 垂直选中浅蓝；Breadcrumb 二级以上；Space/Divider 管间距。

### 3.2 按钮
| 类型 | 何时用 |
|------|--------|
| primary | 每焦点区唯一主 CTA |
| default | 重置/取消/导出 |
| text | 表格行操作 |
| dashed | 添加一项 |
| 危险 | 删除/停用 + 确认 |

主按钮 loading 防重复；图标+文字优先。

### 3.3 表单
筛选 inline/栅格；编辑 vertical 或 horizontal；校验贴字段；整页保存在顶部页头；长表按 Card 分模块。

### 3.4 表格
筛选→工具栏→Table→Pagination；浅灰表头；操作列 fixed right + text 按钮；small 密度；Empty/loading。

### 3.5 反馈层级
| 场景 | 组件 |
|------|------|
| 轻成功/失败/警示（删除完成、保存成功等） | **Message**（见 message-patterns） |
| 较长通知 | Notification |
| 删除确认 | Popconfirm / Modal.confirm |
| 复杂表单 | Drawer 或整页 |
| 阻断说明 | Modal |
| 页内提示 | Alert |
| 加载 | Spin / Skeleton |

### 3.5.1 全局提示 Message（强制结果反馈）

- **用途**：不打断操作的轻量全局反馈；实现用 `Message`。
- **必须出现**：删除成功/失败、保存成功/失败、提交成功/失败、复制成功等操作结果。
- **类型**：success / error / warning / info / normal / loading（按结果等级，勿一律 success）。
- **位置**：默认**顶部居中**；白底+浅描边+轻阴影；左语义图标 + 右 14px 文案。
- **时长**：默认约 3s 自动关闭；loading 持续到结束再切 success/error。
- **链路**：删除 = Popconfirm/Modal 确认 → 成功 `Message.success('删除成功')`；**禁止**成功后再弹成功对话框。
- 细则见 [message-patterns.md](message-patterns.md)。

### 3.6 数据展示
Descriptions 详情；Tag 状态色+文字；Steps 节点连线；Tabs 同级切换；Statistic 小指标；**Progress 百分比**（见 progress-patterns，对齐官方 Progress）；Timeline 日志。

### 3.7 容器
一模块一 Card；Drawer 旁路详情；Dropdown 收纳次要操作。

## 4. 页面类型 → 组件组合

| 页面 | 组合 |
|------|------|
| 工作台 | 横幅/待办 + 指标 + 列表 + 右栏宫格（content-ux） |
| 列表 | Breadcrumb + 筛选 + 工具栏 + Table + Pagination |
| 表单/编辑 | 顶页头操作条 + 分模块 Card + Form |
| 详情 | 顶页头 + 可选 Steps + Descriptions/子表 |
| 设置 | Tabs/锚点 + Form |
| 结果 | Result + 返回 |

---

## 5. 状态必须覆盖
Loading / Empty / Error / 无权限 / 成功反馈。

## 6. 无障碍（轻量）
对比度足够；状态有文字；图标按钮可访问名称；少动效。

## 7. 实现提示
真实组件 API；变量色；顶栏 #004EA2；IconPark；包名仅 import/依赖说明，不进 UI。

## 8. 生成前自检
- [ ] 壳色与菜单选中正确
- [ ] 14px 正文、4 倍间距、圆角≤8
- [ ] 每焦点区一个 primary
- [ ] 反馈层级正确
- [ ] loading/empty
- [ ] 无界面规范注释/库名


## Steps 步骤条宽度（强制）

- 步骤条必须 **充满其所在白色容器/模块内容区的宽度**（`width: 100%`）。
- **保留容器内边距**：由外层 Card / page 模块的 16–20px 内边距承担，步骤条不要再缩成左对齐一小截。
- 布局：节点 `flex: none`；**连接线 `flex: 1`** 均分拉满中间空隙；步骤之间视觉间距靠 line 的水平 margin（约 8–12px）。
- 实现层：`Steps` 放在全宽容器内；勿给 Steps 固定过窄 max-width；水平 `direction=horizontal` 铺满。
- 禁止：步骤条挤在容器左侧、中间大片空白；禁止为了拉满而取消卡片内边距贴边。

## 页面标题与页头操作（canonical）

### 按页面类型

| 页面类型 | 页面标题 | 主操作位置 |
|----------|----------|------------|
| **工作台** | **不要**单独页头标题名（不要「工作台」大标题条） | 内容区内业务按钮；无整页标题行 |
| **列表 / 普通业务页** | **需要**简洁页名（参考「产品列表」：16px / 字重 500，左对齐） | 标题**单独一行**；「新建」等放在**筛选下方、表格上方**工具行右对齐，**禁止与标题左右并列** |
| **详情 / 表单编辑** | **需要** | 页头条（有背景容器）：左标题 + 右返回/取消/保存；页面级操作仅在此条 |

### 列表页推荐结构

```
面包屑（可选）
┌─ 白卡片 ─────────────────────────────┐
│  页面标题（单独一行，无右侧并列按钮）   │
│  筛选区 + 查询 / 重置                  │
│  表格工具行（右：新建等）               │
│  Table + Pagination                    │
└────────────────────────────────────────┘
```

### 禁止

- 工作台再套「工作台」页头标题
- 列表页只有面包屑、没有页面名称
- 列表页标题与「新建」等主按钮同一行左右并列
- 把列表标题做成营销大标题 / 厚重双行英雄区
